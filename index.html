Analise de Requisitos

Funcionais Não Funcionais Inversos

Requisitos Funcioais

  • O sistema deve incentivar a leitura

  • O sistema deve ter um lembrete automático para devoluções

  • O sistema deve recomendar livros baseados no que ja foram lidos

  • O sistema deve oferecer um espaço para discurção sobre os livros

  • O sistema deve registra os livros lidos por mais tempo
  • O sistema deve oferecer um biblioteca digital integrada no sistema escolar
  • O sistema deve oferecer um lugar em que os alunos consigam visualizar quais livros estão disponíveis para reservar
  • O sistema deve permitir que os alunos registrem sugestões de livros

















Requisitos Não Funcioais

  • O diretor poderá monitorar a biblioteca de forma eficiente. Verificando estatísticas dos livros lidos, os alunos que mais acessam o sistema.
  • Somente alunos e professores podem acessar o sistema.
  • Os alunos podem acessar o sistema de qualquer lugar e ler durante intervalos ou no ônibus.
  • Espaço para os professores indicarem leituras personalizadas, levando em consideração a dificuldade e interesses dos alunos
  • Os professores podem anexar atividades complementares aos livros recomendados, como questionários, desafios ou resumos interativos.
  • Os alunos podem reservar um livro antecipadamente
  • Tera um raking dos livros mais lidos pelos alunos da escola.
  • A biblioteca e bem estruturada, contribuindo para o desenvolvimento educacional.

















Requisitos Inversos

  • O sistema não permite atraso de devolução dos livros.
  • O sistema não permite que pessoa de fora da escola acessem o sistema
